                          munificent Mod C Oz • 3 years ago • edited

                          For that case, you won't want to treat them like a pair of nested binary operators. Instead, you'll parse a whole series of them as a single node with a list of operands. Something like:

                          class ComparisonOperatorParselet implements InfixParselet {
                            public Expression parse(Parser parser,
                                Expression left, Token token) {
                              List<expression> operands = new ArrayList<expression>();
                              operands.add(left);
                              operands.add(parser.parseExpression());

                              while (parser.lookAhead().getType() == TokenType.LESS_THAN) {
                                parser.consume();
                                operands.add(parser.parseExpression());
                              }

                              return new ComparisonExpression(operands);
                            }
                          }

                                                                                                  Sam Danielson einsteinsci e • 4 years ago

                                                                                                  This IS the shunting yard algorithm. It's succinct because it's using the call stack instead of an explicit one. Here is the key insight: "If [the parselet] encounters an expression whose precedence is lower than we allow, it just stops parsing and returns what it has so far." In other words, the parslet can either push, which is to recursively call parslets, or pop, which is to return a value.
